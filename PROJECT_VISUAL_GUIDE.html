APP.BUENA.CONTEXT

One living context for property management.

A hackathon prototype that turns scattered bank records, invoices, emails, letters, and master data into one durable, editable context artifact.

Demo Ready Flow

Context Engine

context.md

A single living artifact that the property manager can ask, edit, protect, and update.

READ

Ask Questions

The chatbot routes to a building, retrieves evidence, attaches citations, then lets Academic Cloud synthesize a natural answer.

WRITE

Add Context

Chat write commands and resource intake create guarded patches instead of editing files randomly.

PROTECT

Human Truth

Manual corrections are wrapped in <user> tags and preserved across future ingestion.

RECOVER

Rollback

Every write can be audited. Rollback shows a diff before restoring an older snapshot.

Chat Question Flow

01Route Building

Selects the relevant property context, usually LIE-001 for the demo.

02Search Context

Finds the strongest sections and quotes from context.md.

03Model Synthesis

Academic Cloud writes the final answer from retrieved evidence only.

04Citations

Returns source section names and snippets for trust.

05Visual Trace

The UI shows the route/search/model/citation path.

Write And Intake Flow

01User Input

Paste a resource or type a command like Add note:.

02Validation

Reject spam/noise and route to an allowed context section.

03Dry-run Diff

Preview before/after changes before applying resource writes.

04Guarded Write

Preserve protected user blocks and append an audit event.

05Highlight

Refresh artifact and scroll to the inserted context.

Safety Gates

  • Role gates: viewer reads, editor previews, approver writes, admin rolls back.
  • User block guard: any changed <user> block blocks the write.
  • Diff preview: resource and rollback flows show changes before apply.
  • Audit log: writes keep before/after snapshots for recovery.
  • Evidence boundary: model answers must use retrieved context only.
